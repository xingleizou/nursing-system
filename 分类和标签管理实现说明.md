## 数据库结构（已存在）

### 1. 分类表 (`knowledge_category`)
```sql
CREATE TABLE knowledge_category (
  category_id BIGINT PRIMARY KEY AUTO_INCREMENT,
  parent_id BIGINT DEFAULT 0 COMMENT '父分类ID',
  category_name VARCHAR(100) NOT NULL COMMENT '分类名称',
  order_num INT DEFAULT 0 COMMENT '排序',
  is_system CHAR(1) DEFAULT '0' COMMENT '是否系统内置',
  status CHAR(1) DEFAULT '0' COMMENT '状态',
  create_by VARCHAR(50) COMMENT '创建者',
  create_time DATETIME COMMENT '创建时间',
  update_by VARCHAR(50) COMMENT '更新者',
  update_time DATETIME COMMENT '更新时间',
  remark VARCHAR(500) COMMENT '备注',
  deleted INT DEFAULT 0 COMMENT '删除标志'
);
```

### 2. 标签表 (`knowledge_tag`)
```sql
CREATE TABLE knowledge_tag (
  tag_id BIGINT PRIMARY KEY AUTO_INCREMENT,
  tag_name VARCHAR(100) NOT NULL COMMENT '标签名称',
  color VARCHAR(20) COMMENT '标签颜色',
  status CHAR(1) DEFAULT '0' COMMENT '状态',
  create_by VARCHAR(50) COMMENT '创建者',
  create_time DATETIME COMMENT '创建时间',
  deleted INT DEFAULT 0 COMMENT '删除标志'
);
```

### 3. 资源标签关联表 (`knowledge_resource_tag`)
```sql
CREATE TABLE knowledge_resource_tag (
  resource_id BIGINT NOT NULL COMMENT '资源ID',
  tag_id BIGINT NOT NULL COMMENT '标签ID',
  PRIMARY KEY (resource_id, tag_id)
);
```

## 关联关系

### 1. 资源与分类
- **一对一关系**：一个资源属于一个分类
- **字段关联**：`knowledge_resource.category_id` → `knowledge_category.category_id`
- **业务规则**：
  - 上传资源时必须选择分类
  - 分类删除时需要检查是否有资源使用

### 2. 资源与标签
- **多对多关系**：一个资源可以有多个标签，一个标签可以标记多个资源
- **中间表**：`knowledge_resource_tag`
- **业务规则**：
  - 上传资源时可以选多个标签（可选）
  - 标签删除时需要解除与资源的关联

## 使用流程

### 1. 管理员操作流程
1. **创建分类**：在分类管理页面创建分类（如：制度规范、操作手册、论文资料）
2. **创建标签**：在标签管理页面创建标签（如：重要、紧急、培训资料）
3. **上传资源**：在资源管理页面选择分类和标签后上传文件
4. **管理资源**：审核、预览、下载、删除资源

### 2. 用户操作流程
1. **浏览资源**：按分类或标签筛选资源
2. **预览/下载**：查看或下载已发布的资源
3. **搜索资源**：按标题、分类、标签搜索

## 注意事项

1. **分类的树形结构**：需要递归处理父子关系
2. **标签的颜色**：前端需要支持颜色选择，后端存储颜色值
3. **数据一致性**：删除分类或标签时需要处理关联的资源
4. **权限控制**：分类和标签管理应该是管理员专属功能
5. **用户体验**：分类和标签选择应该支持搜索和快速创建
